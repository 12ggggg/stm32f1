基于 STM32F103 的多功能嵌入式系统
本项目基于野火 STM32F103 开发板，实现了一个集 SD 卡音乐播放、GPS 定位、WiFi 天气获取于一体的多功能嵌入式系统。系统采用 FreeRTOS 多任务架构，独立完成 SDIO 底层驱动、VS1053 音频解码、GPS NMEA 解析、ESP8266 AT 指令联网，并移植 LVGL 图形界面，在 64KB RAM 的极限资源下通过精细的内存裁剪与动态链表菜单实现了流畅的交互体验。

功能特点
SD 卡音乐播放

自研 SDIO 驱动，支持 1bit/4bit 模式，兼容 SDSC/SDHC 卡

DMA 多块读写，FATFS 文件系统，支持 MP3/WAV 格式

VS1053 音频解码：音量调节、高低音均衡、多倍速播放（1x/2x/4x）

GPS 定位

USART2 + DMA + 空闲中断接收 NMEA 语句

解析 $GNRMC/$GNGGA，提取 UTC 时间、经纬度、海拔、卫星数

双缓冲机制，中断只存原始帧，任务内解析，防止丢帧

WiFi 天气获取

ESP8266 AT 指令封装，自动配网、断线重连

TCP 连接心知天气 API，HTTP GET 请求，自研 JSON 解析

支持多城市切换，手动/自动刷新

LVGL 图形界面

240×320 触摸屏，FSMC 驱动 ILI9341，XPT2046 触摸

动态菜单系统：链表驱动，进出页面时释放并重建，内存按需使用

状态栏实时显示 WiFi/GPS/SD 状态，音乐播放界面支持进度条、音量调节、倍速切换

FreeRTOS 多任务架构

6 个任务：UI 渲染、GPS 解析、WiFi 联网、SD 卡管理、音乐播放、主控调度

任务通知解耦数据采集与 UI 刷新，互斥量保护 LVGL 资源

优先级抢占：音乐任务 4、UI 任务 5、GPS 任务 3

极限内存优化

在 64KB RAM 下，经历多次 HardFault 后定位任务栈溢出、数组越界、临界区竞争

功能裁剪：FreeRTOS 禁用非必要组件，LVGL 关闭动画、平滑滚动，双缓冲改为单缓冲+局部刷新

精细划分：FreeRTOS 堆 24KB、LVGL 缓冲 15KB、系统堆剩余，稳态 RAM 占用 56KB，长期运行无泄漏
